# 项目实战与复盘

> 项目页的重点不是堆功能，而是把知识点放进真实链路，说明设计选择、失败点和评测方式。

## 一、当前项目主线

| 项目方向 | 重点能力 | 当前入口 |
| :--- | :--- | :--- |
| 企业知识库 Agent | 文档 ingestion、检索、引用、权限与评测 | [复盘模板](01_知识库Agent复盘模板.md) |
| 工具型 Workflow Agent | Tool Schema、状态流转、HITL、失败恢复 | [复盘模板](02_工具型WorkflowAgent复盘模板.md) |
| Mini-SWE Code Agent | 代码工具、patch、测试回环、失败分类 | 后续补充 |

## 二、复盘一个项目时看什么

1. **输入输出**：用户给什么，系统返回什么。
2. **主链路**：节点、数据、工具和状态如何流动。
3. **关键难点**：哪里最容易错，为什么。
4. **工程兜底**：超时、重试、安全、回滚、人工确认。
5. **评测指标**：如何证明改动有效。
6. **面试追问**：如何解释设计取舍。
