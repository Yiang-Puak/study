# 03 | RAG 检索增强生成 —— 核心概念与面试答题模板

## 一、核心概念速记

### 1. RAG 四阶段流程（⭐⭐⭐⭐⭐ 必会）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  索引   │ → │  检索   │ → │  增强   │ → │  生成   │
│ Indexing│    │Retrieve │    │ Augment │    │ Generate│
└────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘
     │              │              │              │
  文档加载       语义检索/        将检索到的      LLM 结合
  Chunk 切分     关键词检索       上下文拼入      上下文生成
  Embedding      向量相似度       Prompt         最终答案
  向量入库       混合排序
```

**面试时能画出这个图，印象分拉满。**

### 2. Embedding 层优化（⭐⭐⭐⭐）

| 优化点 | 说明 |
|--------|------|
| **模型选型** | BGE（百度）、E5（微软）、GTE（阿里）、text-embedding-v4（通义），中文场景 BGE-M3 效果较好 |
| **领域微调** | 通用 Embedding 在专业领域（金融/法律/医疗）效果差，需用领域语料 fine-tuning |
| **Chunk 策略** | 固定长度（如 512 token）/ 按语义切分 / 按段落切分 / 重叠滑动窗口；面试常问：Chunk 太大检索粒度粗，太小上下文割裂 |
| **多粒度索引** | 同时索引句子级（精细）和段落级（完整），检索时融合两者结果 |

### 3. 检索层优化（⭐⭐⭐⭐⭐ 超高频）

| 技术 | 原理 | 面试关键词 |
|------|------|------------|
| **混合检索** | 语义检索（向量相似度）+ BM25 关键词检索，两者结果融合 | Hybrid Search、α 权重 |
| **动态 α 调整** | 实体类查询（如"某公司法条"）α=0.3（关键词权重高）；分析类查询（如"总结趋势"）α=0.7（语义权重高） | 路由/分类模型决定 α |
| **重排序 Reranker** | 先召回 top_100，再用轻量模型（如 bge-reranker）精排取 top_5 | 两阶段检索、精排 |
| **查询改写** | LLM 先扩展/改写用户 query，再用改写后的 query 检索 | Query Expansion、HyDE |

### 4. 缓存优化（⭐⭐⭐）

| 层级 | 实现 | 用途 |
|------|------|------|
| **内存缓存** | Python dict / LRU Cache | 热点 query，微秒级响应 |
| **Redis 缓存** | Key=query_hash, Value=answer | 跨会话共享，秒级 TTL |
| **磁盘缓存** | 本地文件 / SQLite | 离线场景，防重复生成 |
| **失效策略** | 基于 TTL + 内容变化检测 | 知识库更新时主动失效 |

### 5. 评估指标（⭐⭐⭐⭐）

| 指标 | 含义 | 适用场景 |
|------|------|----------|
| **命中率 Hit Rate** | 正确答案是否在检索结果中 | 检索阶段 |
| **MRR** | 第一个正确答案的排名倒数均值 | 检索阶段 |
| **NDCG** | 考虑排序位置的相关性加权得分 | 检索阶段 |
| **精确率/召回率** | 检索结果中相关文档比例 / 所有相关文档被检索到的比例 | 检索阶段 |
| **答案正确率** | LLM 生成答案的事实正确性 | 生成阶段 |
| **忠实度 Faithfulness** | 答案是否忠实于检索到的上下文（有无 hallucination） | 生成阶段 |

## 二、面试真题 + 标准答案（五步结构法）

### 【真题 1】RAG 的完整流程是什么？画出链路图（⭐⭐⭐⭐⭐）

**答题模板**：
1. **核心定义**：RAG（Retrieval-Augmented Generation）= 检索 + 增强 + 生成，让 LLM 在回答前先查资料，减少幻觉。
2. **与纯 LLM 的区别**：纯 LLM 依赖训练参数中的知识（有截止日期、会幻觉）；RAG 让 LLM 动态访问外部知识库，答案更实时、可溯源。
3. **四阶段流程**：
   - **索引**：加载文档 → Chunk 切分 → Embedding → 存入向量库（Chroma/Milvus）
   - **检索**：用户 query Embedding → 向量相似度搜索 → 召回 top_k 相关 chunk
   - **增强**：将召回的 chunk 拼入 Prompt（如"基于以下资料回答：{context}"）
   - **生成**：LLM 结合上下文生成答案
4. **常见坑点**：Chunk 切分不当导致语义割裂；检索召回不相关文档会污染生成结果；上下文过长超出模型限制。
5. **生产优化**：混合检索（语义+关键词）+ Reranker 精排 + 多级缓存 + 检索结果溯源（告诉用户答案来自哪篇文档）。

---

### 【真题 2】Embedding 怎么选？Chunk 怎么切？（⭐⭐⭐⭐）

**答题模板**：
1. **核心定义**：Embedding 是将文本映射到语义向量的过程，Chunk 是将长文档切分为适合检索和生成的片段。
2. **选型原则**：
   - 中文通用场景：BGE-M3、GTE-large（阿里）
   - 英文通用场景：E5、OpenAI text-embedding-3
   - 专业领域：必须在领域语料上微调（如金融文本用金融语料 fine-tune）
3. **Chunk 策略**：
   - **固定长度**：按 512/1024 token 切，简单但有语义割裂风险
   - **按段落/句子**：保留自然边界，粒度粗
   - **重叠窗口**：相邻 Chunk 有 overlap，减少边界信息丢失
   - **语义切分**：用模型判断语义断点，效果较好但成本高
4. **常见坑点**：Chunk 太大 → 检索粒度粗，召回精准度低；Chunk 太小 → 缺少上下文，LLM 生成质量差。
5. **生产优化**：多粒度索引（句子级 + 段落级同时建索引）；Chunk 内加入标题/章节信息增强语义；对高频查询做 Embedding 结果缓存。

---

### 【真题 3】什么是混合检索？α 怎么调？（⭐⭐⭐⭐⭐）

**答题模板**：
1. **核心定义**：混合检索 = 语义检索（向量相似度）+ 关键词检索（BM25），两者结果融合后返回。
2. **为什么需要**：纯语义检索对专业术语/实体匹配不准（如"Transformer"在语义空间可能和"变压器"混淆）；纯关键词检索无法理解同义词和语义相近表达。
3. **融合公式**：`score = α * vector_score + (1-α) * bm25_score`，α ∈ [0, 1]
4. **α 动态调整**：
   - 实体查询（查具体人名/法条/产品型号）：α 偏小（0.2-0.4），关键词更准
   - 分析查询（总结/对比/解释）：α 偏大（0.6-0.8），语义理解更重要
   - 可用一个小分类器判断 query 类型，自动路由 α 值
5. **生产优化**：先做路由分类，再决定 α；对融合后的结果再用 Reranker 精排；记录不同 α 下用户满意度，A/B 测试调优。

---

### 【真题 4】Reranker 是什么？为什么需要两阶段检索？（⭐⭐⭐⭐）

**答题模板**：
1. **核心定义**：Reranker 是检索后的精排模型，对召回的大量候选结果重新打分排序。
2. **为什么需要两阶段**：
   - 第一阶段（召回）：要求速度快，用轻量方法（向量检索、BM25）召回 top_100
   - 第二阶段（精排）：要求精准度高，用更强的交叉编码器（Cross-Encoder）对 query-doc 做精细交互打分，取 top_5/10
   - 如果直接用精排模型做召回，计算量太大（O(N) 的交叉注意力）
3. **典型模型**：bge-reranker、Cohere Rerank、Jina Reranker
4. **常见坑点**：Reranker 也有幻觉风险——给不相关文档打高分；精排模型输入长度有限，长文档需要截断。
5. **生产优化**：Reranker 结果加入置信度阈值，低于阈值的文档丢弃；Reranker 结果也缓存；对高频 query 的精排结果做离线预计算。

---

### 【真题 5】RAG 怎么评估效果？（⭐⭐⭐⭐）

**答题模板**：
1. **核心定义**：RAG 评估分两个阶段——检索质量评估 + 生成质量评估。
2. **检索阶段指标**：
   - 命中率（Hit@K）：正确答案是否在 top_k 中
   - MRR：第一个正确答案的排名倒数，关注"多快找到"
   - NDCG：考虑排序位置的相关性加权，关注"排得对不对"
   - 精确率/召回率：传统 IR 指标
3. **生成阶段指标**：
   - 答案正确率：人工或强模型评判答案事实正确性
   - 忠实度（Faithfulness）：答案是否基于检索上下文，而非 LLM 自身知识
   - 上下文相关性：检索到的 chunk 是否与 query 相关
4. **常见坑点**：没有 ground truth 时难以自动化评估；人工评估成本高；不同领域标准不一致。
5. **生产优化**：建立标注数据集（query + 期望答案 + 相关文档）；用 LLM-as-a-Judge（GPT-4 自动评分）；A/B 测试对比不同检索策略的用户满意度。
