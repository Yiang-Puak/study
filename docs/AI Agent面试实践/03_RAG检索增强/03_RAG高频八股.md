## 1. 什么是 RAG

**一句话答案：**RAG 是让模型在回答前先检索外部证据，再基于证据组织回答的生成增强链路。

**30 秒展开：**

1. 检索层从知识库召回与问题相关的片段。
2. 上下文层把证据、引用信息和回答约束放进 Prompt。
3. 生成层输出答案、引用或拒答。
4. 工程价值是降低知识陈旧和无依据回答的风险，但它不保证天然正确。

## 2. RAG 完整链路是什么

**一句话答案：**离线做索引，在线做检索、增强和生成，最后用评测与 Trace 回流优化。

```text
文档接入 -> 清洗与切分 -> Embedding 与索引
用户问题 -> Query 处理 -> 召回与过滤 -> Rerank
证据组织 -> 回答生成 -> 引用 / 拒答 -> 评测
```

**记忆口诀：**先建证据库，再找证据，再摆证据，再答证据。

## 3. Chunk 太大或太小会怎样

**一句话答案：**太大导致检索粒度粗和噪声多，太小导致语义断裂和证据不完整。

**答题要点：**

- 优先保留标题、段落、表格说明等结构边界。
- 用 overlap 降低切分边界损失，但 overlap 不是越大越好。
- 对长文档可结合父子块、多粒度索引和 metadata 过滤。

## 4. Hybrid Retrieval 和 Reranker 怎么区分

| 能力 | 解决的问题 | 面试表述 |
| :--- | :--- | :--- |
| Hybrid Retrieval | 召回阶段别漏掉实体词或同义表达 | 关键词检索与语义检索互补 |
| Reranker | 候选已经找到了，谁更相关 | 在候选集上做更精细排序 |

**一句话答案：**Hybrid 先拓宽可靠召回面，Reranker 再提高候选排序质量。

## 5. RAG 为什么不能完全消除幻觉

**一句话答案：**因为 RAG 只把证据送到模型面前，证据是否正确、上下文是否清晰、模型是否忠实归纳仍会出错。

**工程补充：**

- 检索不到证据时要能拒答。
- 引用要绑定实际证据片段，不能只拼一个来源名。
- 评测要区分召回失败、排序失败、上下文污染和生成失败。

## 6. RAG 怎么评测

| 层次 | 重点 |
| :--- | :--- |
| 检索 | 正确证据是否进候选集，相关片段是否排在前面 |
| 生成 | 答案是否正确、是否忠实于证据、是否该拒答时拒答 |
| 体验 | 延迟、引用可读性、答案覆盖度 |

**答题顺序：**先说分层，再说指标，再说误差分析。
