> 八股页负责把知识压缩成可复述答案；专题页负责把答案背后的原理讲明白。

## 一、八股题怎么刷

推荐按三轮走：

1. **第一轮说定义**：每题先用一句话答清楚。
2. **第二轮补原理**：加入流程、边界和常见坑点。
3. **第三轮加项目举证**：把答案落到 RAG、Tool、SQL、安全或评测链路。

## 三、标准答题骨架

一题尽量压成下面五层：

```text
1. 一句话定义
2. 与相近概念的边界
3. 核心流程或原理
4. 生产中的坑点
5. 项目或工程优化
```

## 四、八股示例

### 题目：RAG 为什么不能完全消除幻觉？

**一句话答案：**RAG 负责提供外部证据，但召回、上下文组织和生成忠实度都可能出错。

**展开回答：**

- 检索阶段可能没有召回正确片段，或者召回了噪声。
- Chunk 边界、元数据过滤和 rerank 质量会影响证据质量。
- 生成阶段即使看到证据，也可能错误归纳或过度补全。
- 工程上要拆开评测召回质量和回答质量，再配合引用、拒答与 Trace 排查。

**追问方向：**

- 怎么判断是检索错还是生成错？
- 你会用哪些 RAG 指标？
- 什么场景要加 reranker？
