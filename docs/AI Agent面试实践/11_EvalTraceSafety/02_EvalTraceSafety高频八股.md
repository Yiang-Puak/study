# Eval、Trace 与 Safety 高频八股

## 题单速刷

| 题目 | 一句话先答 |
| :--- | :--- |
| 为什么 Agent 需要 Eval | 因为一次 Demo 不能证明改动稳定有效 |
| Trace 要记录什么 | 目标、节点、证据、工具、结果、重试和决策 |
| Prompt 是安全边界吗 | 不是，边界在执行层 |
| RAG 与 Tool 怎么分层评测 | 检索、工具、任务、系统分层 |
| HITL 什么时候需要 | 高风险、副作用大、低置信动作 |
| 怎么做失败分类 | 按检索、规划、工具、生成、安全拆层 |

## 1. 为什么 Agent 需要 Eval

**一句话答案：**Agent 行为由模型、上下文、检索和工具共同决定，必须用固定 case 和指标做回归，才能判断优化是否真的有效。

## 2. Trace 和普通日志的区别

**一句话答案：**日志记录事件，Trace 要保留跨节点调用链和关键决策上下文，帮助定位失败发生在哪一层。

## 3. Prompt 为什么不是安全边界

**一句话答案：**Prompt 只能影响模型行为，不能替代权限、校验、审批、隔离和审计。

## 4. 指标怎么分层

| 层 | 重点 |
| :--- | :--- |
| Retrieval | 证据召回 |
| Tool | 选择和参数 |
| Task | 是否完成 |
| System | 延迟、成本、稳定性 |
| Safety | 是否按策略拦截 |
