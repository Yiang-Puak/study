# Tool 与 MCP 真题与工程追问

## 一、题型分层

| 层次 | 题目样式 | 主要考察 |
| :--- | :--- | :--- |
| 八股题 | Tool、Function Calling、MCP 分别是什么 | 概念边界 |
| 工程题 | 工具超时、重复写入、权限越界怎么办 | 可靠性与安全 |
| 系统题 | 为什么要把多个外部能力做成 MCP Server | 标准化与复用 |
| 追问题 | 工具返回脏数据后模型继续错下去怎么办 | 观测、校验、收口 |

## 二、示范真题

### 题 1：模型决定调用删除类工具，你怎样保证它不会误删

**回答骨架：**

1. 模型只生成调用意图，不直接拿到无边界执行权。
2. 工具输入要有 schema、路径或资源范围限制、权限校验。
3. 删除类动作加 preview、人工确认、审计和可恢复策略。
4. 运行时记录调用 Trace、操作者、参数、结果和失败原因。

### 题 2：工具调用经常超时，Agent 还要继续完成任务，你怎么设计

**回答骨架：**

1. 区分可重试与不可重试错误。
2. 设置 timeout、指数退避和最大重试次数。
3. 写操作加入幂等键或版本检查。
4. 提供 fallback：换数据源、返回部分结果、询问用户补充条件。
5. 用延迟、错误率和重试次数做监控。

### 题 3：已有一套内部 API，为什么还要考虑 MCP

**先答结论：**不是为了替代 API，而是为了让 AI Host 以统一方式发现和使用工具、资源与提示词。

**继续展开：**

- API 仍是业务能力本体。
- Tool 包装负责描述、参数和结果契约。
- MCP 负责 Host、Client、Server 之间的标准连接与能力暴露。
- 接入前仍要评估权限、网络边界和维护成本。

## 三、工程追问清单

| 追问 | 回答提示 |
| :--- | :--- |
| 工具描述写得很像，模型总选错怎么办 | 拆清职责、补正反例、缩小重叠能力 |
| 工具返回超长文本怎么办 | 结构化提取、分页、摘要、资源引用 |
| 什么时候要让工具直接返回错误而不是让模型猜 | 参数非法、权限不足、高风险失败 |
| 本地 MCP Server 最大风险是什么 | 文件、命令、凭据和权限越界 |
| 多个 Server 能力重名怎么办 | 命名空间、能力说明、策略约束 |
| 怎样把工具调用纳入评测 | 成功率、参数正确率、步骤数、失败恢复 |
