# 06 | 模型微调与性能优化

> 面试手册 1.6 节补充

## 一、核心概念速记

### 1. 参数高效微调 PEFT（⭐⭐）

**为什么需要 PEFT**：
- 全量微调 70B 参数模型需要数百 GB 显存，成本极高
- PEFT 只微调少量参数，就能达到接近全量微调的效果

| 方法 | 原理 | 特点 |
|------|------|------|
| **LoRA** | 在权重矩阵旁添加低秩分解矩阵（A×B） | 参数量最小（千分之一），效果接近全量微调 |
| **QLoRA** | LoRA + 4-bit 量化 | 单卡可微调 65B 模型 |
| **P-Tuning** | 添加可训练的提示嵌入 | 适合分类/抽取任务 |

**面试金句**：LoRA 的核心是"冻结原权重，只训练低秩旁路"，推理时可以将旁路合并回主权重，零推理开销。

### 2. 推理加速（⭐⭐）

| 技术 | 原理 | 效果 |
|------|------|------|
| **量化 Quantization** | 权重从 FP32/FP16 降到 INT8/INT4 | 模型体积和显存占用大幅减少 |
| **知识蒸馏** | 大模型（教师）教小模型（学生） | 小模型达到大模型 90% 效果 |
| **Flash Attention** | 优化注意力计算的内存访问模式 | 长序列推理速度提升 2-4 倍 |
| **vLLM / TensorRT-LLM** | 批处理优化和 kernel 融合 | 吞吐量提升数倍 |

### 3. Agent 后训练（⭐⭐）

**Tool-Use 微调**：
- 用包含工具调用轨迹的数据集训练模型
- 让模型学会"什么时候该调用工具、怎么填参数"

**RLHF / DPO**：
- **RLHF**：人类标注偏好 -> 训练奖励模型 -> 用 PPO 强化学习优化
- **DPO**：直接用偏好数据优化，省去奖励模型，更简单高效
- **目的**：让模型输出更符合人类期望（有用、无害、诚实）

## 二、面试真题 + 标准答案

### 【真题 1】LoRA 的核心原理是什么？为什么能节省显存？（⭐⭐）

**答题模板**：
1. **核心定义**：LoRA（Low-Rank Adaptation）是一种参数高效微调方法，通过在原始权重矩阵旁添加低秩分解矩阵来实现微调。
2. **原理**：假设原始权重 W 不变，引入 W' = W + BA，其中 B 和 A 是小矩阵（秩 r 远小于维度 d）。微调时只训练 B 和 A，W 保持冻结。
3. **为什么省显存**：全量微调需要存储和更新所有参数（如 7B 模型需要 28GB+ 显存）；LoRA 只更新少量参数（如 r=8 时只需要更新约 0.1% 的参数）。
4. **常见坑点**：LoRA 的秩 r 选择很重要，r 太小表达能力不足，r 太大失去效率优势；LoRA 不适合需要大幅改变模型行为的场景。
5. **生产优化**：对不同层应用不同秩（注意力层用较大 r，FFN 层用较小 r）；推理时将 BA 合并回 W，避免额外计算开销。

---

### 【真题 2】模型量化是什么？INT4 和 FP16 有什么区别？（⭐⭐）

**答题模板**：
1. **核心定义**：量化是将模型权重从高精度（如 FP32）映射到低精度（如 INT8/INT4）表示的过程，目的是减少模型体积和显存占用。
2. **FP16 vs INT4**：FP16 是半精度浮点（16位，有符号位+指数位+尾数位），INT4 是 4 位整数（只能表示 16 个离散值）。INT4 将权重压缩到原来的 1/8，但精度损失更大。
3. **实现方式**：训练后量化（PTQ）直接对训练好的模型量化；量化感知训练（QAT）在训练时就模拟量化效果，精度更好。
4. **常见坑点**：量化对注意力层影响较大，可能降低生成质量；INT4 在某些任务上精度下降明显；需要配合校准数据集做动态量化。
5. **生产优化**：混合量化（敏感层用 FP16，不敏感层用 INT4）；AWQ/GPTQ 等先进量化算法在保持精度的同时实现高压缩比。

---

### 【真题 3】RLHF 和 DPO 有什么区别？（⭐⭐）

**答题模板**：
1. **核心定义**：RLHF（Reinforcement Learning from Human Feedback）和 DPO（Direct Preference Optimization）都是让模型输出对齐人类偏好的训练方法。
2. **RLHF 流程**：收集人类偏好数据 -> 训练奖励模型（Reward Model）-> 用 PPO 强化学习优化策略模型。流程复杂，需要维护多个模型。
3. **DPO 流程**：直接用偏好数据优化，跳过奖励模型，将问题转化为分类损失。公式本质是在偏好数据上最大化似然差。
4. **对比**：RLHF 更灵活但实现复杂、训练不稳定；DPO 更简单高效，但依赖偏好数据质量。工业界 DPO 逐渐成为主流。
5. **生产优化**：先用 SFT（监督微调）做预热，再用 DPO 做偏好对齐；定期用新的人类反馈数据更新；多轮迭代逐步提升对齐度。
