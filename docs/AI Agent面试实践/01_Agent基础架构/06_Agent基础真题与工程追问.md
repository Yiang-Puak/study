# Agent 基础真题与工程追问

## 一、示范真题

### 题 1：你做的是 Agent，还是一个固定 LLM 工作流

**回答骨架：**

1. 先定义两者边界：Workflow 固定步骤，Agent 根据观察动态决策。
2. 说明系统里哪些节点是固定的，哪些节点允许模型选动作。
3. 说明为什么这么切：可靠性、成本、安全和任务开放度。
4. 用 Trace 或 Eval 证明这个设计不是纯概念。

### 题 2：Agent 怎么避免无限循环和错误行动

**回答骨架：**

1. 系统要有最大步数、超时和退出条件。
2. 工具调用做 schema、权限和失败分类。
3. 低置信或高风险动作进入人工确认。
4. 失败样本进入评测集，持续回归。

### 题 3：为什么不是所有复杂任务都拆成 Multi-Agent

**回答骨架：**

1. 多 Agent 增加通信、状态同步和冲突裁决。
2. 若任务强耦合，拆角色反而增加上下文损耗。
3. 先做单 Agent 或 Workflow 基线，再用评测看拆分收益。

## 二、工程追问清单

| 追问 | 回答提示 |
| :--- | :--- |
| Agent 的停止条件在哪里 | 模型停止意图加系统上限 |
| 规划失败怎么修 | 约束任务、补观察、引入检查节点 |
| 为什么要有 Observation | 行动结果必须回流决策 |
| 如何描述项目里的 Agent 性 | 动态工具选择、状态反馈、失败恢复 |
