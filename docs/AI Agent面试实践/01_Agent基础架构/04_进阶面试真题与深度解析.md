# 04 | Agent 基础架构 —— 进阶面试真题与深度解析

## 真题 5：工具调用的容错机制怎么设计？（⭐⭐）

**答题模板**：
1. **核心定义**：工具调用容错是指当外部工具执行失败（超时、返回错误、网络中断）时，Agent 能优雅降级而非直接崩溃。
2. **为什么必须做**：生产环境工具不可靠（第三方 API 限流、服务宕机），没有容错 = Agent 用户体验极差。
3. **实现机制**：
   - **置信度阈值过滤**：LLM 对 tool_call 的置信度低于阈值时，要求重新推理或拒绝执行
   - **异常重试队列**：工具失败时自动重试（指数退避），记录重试次数，超过上限则放弃
   - **人工干预降级通道**：关键操作前加入 Human-in-the-loop 确认；工具全部失败时降级为纯 LLM 回答
4. **常见坑点**：重试可能导致重复操作（如重复扣款），需保证工具幂等性；异常信息不要直接暴露给用户，需包装为友好提示。
5. **生产优化**：接入监控告警（Prometheus + Grafana），统计工具成功率、延迟、错误类型；对高频失败工具做熔断（Circuit Breaker）。

## 真题 6：如何设计可复用的 Agent Skill 库？（⭐⭐⭐）

**答题模板**：
1. **核心定义**：Skill = 名称 + 描述 + 输入参数定义（JSON Schema）+ 执行逻辑 + 输出格式规范。
2. **设计原则**：
   - **单一职责**：每个 Skill 只做一件事（如"查天气"不做"订机票"）
   - **自描述**：描述要足够清晰，让 LLM 能判断什么时候该调用它
   - **参数严格**：用 JSON Schema 约束参数类型、必填项、取值范围
3. **代码示例**：
   ```python
   @skill(
       name="get_weather",
       description="查询指定城市的当前天气",
       params={
           "type": "object",
           "properties": {
               "city": {"type": "string", "description": "城市名"}
           },
           "required": ["city"]
       }
   )
   def get_weather(city: str) -> str:
       ...
   ```
4. **常见坑点**：描述写得模糊导致 LLM 误调用（如"搜索"不明确是搜网页还是搜数据库）；参数命名不一致（`city` vs `location`）。
5. **生产优化**：Skill 注册中心统一管理；版本控制（Skill 更新后向后兼容）；Skill 调用日志用于后续优化描述文案。

## 真题 7：你有没有了解过 Harness 工程？主要难点是什么？

**答题模板**：
1. **核心定义**：Harness 工程（也叫 Agent 后训练 / Agent 对齐）是指训练 LLM 更好地执行 Agent 任务（工具选择、参数填充、推理链生成）。
2. **为什么重要**：基座 LLM 原生不擅长多步推理和工具调用，需要专门的数据集和训练方法提升 Agent 能力。
3. **主要难点**：
   - **数据收集难**：高质量的 Agent 轨迹数据（thought-action-observation 完整链路）标注成本极高
   - **工具泛化**：训练时见过的工具有限，遇到新工具时泛化能力差
   - **错误传播**：一步推理错误会导致后续全错，需要设计纠错机制
   - **评估困难**：Agent 任务成功标准模糊（"帮我订机票"——什么价位？什么时间算成功？）
4. **解决方案方向**：
   - Tool-Use 微调（专门数据集训练工具调用能力）
   - RLHF / DPO 对齐（让 LLM 学会"什么时候该停"）
   - 合成数据生成（用强模型生成训练轨迹，蒸馏给弱模型）

## 真题 8：Agent 的 Token 消耗怎么优化？（⭐⭐⭐）

**答题模板**：
1. **问题本质**：Agent 多轮循环 + 历史消息累积 → Token 随轮数线性增长 → 成本和延迟双双爆炸。
2. **优化方案**：
   - **历史消息摘要压缩**：每 N 轮将对话历史压缩为摘要，替换原始消息
   - **分层记忆**：短期记忆只保留最近 K 轮；长期记忆用向量库存储，需要时检索召回
   - **工具结果截断**：API 返回结果往往很长，只取前 M 个字符或结构化提取关键字段
   - **限制最大循环次数**：max_iterations 设为合理值（如 5-10）
3. **常见坑点**：过度压缩导致上下文丢失关键信息；摘要生成本身也消耗 Token。
4. **生产优化**：对高频任务预缓存工具结果；用更小的模型做初步筛选（小模型决定要不要调工具，大模型做最终生成）。
