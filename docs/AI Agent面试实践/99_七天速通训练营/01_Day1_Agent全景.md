## 10 分钟知识卡片

### 1. Agent 是什么

一句话定义：

> Agent 是能基于目标、上下文和工具，在多步任务中自主决策并执行动作的系统。

普通聊天机器人主要输出文本；Agent 不只回答，还会调用工具、读取状态、执行动作、观察结果并继续下一步。

### 2. Agent 五层架构

```text
模型层：LLM / Token / 结构化输出
上下文层：Prompt / Memory / RAG
工具层：Tool Calling / Function Schema / MCP
编排层：LangGraph / Planner / Workflow / Multi-Agent
治理层：Eval / Trace / Safety / HITL
```

### 3. 项目举证

工具型 Agent 中：

- **模型层**：LLM 负责理解任务和生成结构化决策；
- **上下文层**：检索证据 + 多轮 history；
- **工具层**：搜索、文件或业务 API；
- **编排层**：工作流状态机；
- **治理层**：权限、Trace、Eval Harness。

## 15 分钟项目举证

请你口述下面这段：

> 工具型 Agent 不是普通问答机器人。它接收到任务后，会整理上下文、选择工具、执行动作、处理失败并返回可追踪结果。整个过程需要编排、权限边界和评测，而不是只让模型生成一段文本。

## 20 分钟面试训练

??? question "Q1：Agent 和普通聊天机器人有什么区别？"
    **答题结构：**

    1. 普通聊天机器人主要生成文本；
    2. Agent 能基于目标和上下文调用工具并执行动作；
    3. Agent 需要状态、工具、编排、观测和安全；
    4. 用项目举例：它会检索证据、选择工具、执行动作、失败重试并返回结构化结果。

??? question "Q2：你怎么理解 AI Agent 系统架构？"
    **30 秒答案：**

    我会按五层理解：模型层提供推理和生成能力；上下文层决定模型看到什么，包括 Prompt、Memory 和 RAG；工具层让模型连接外部系统；编排层用 LangGraph 或 Planner 管理多步流程；治理层负责评测、可观测、安全和人工审批。真正的 Agent 不是一个 Prompt，而是一个可执行、可观测、可评测的系统。

??? question "Q3：为什么不是所有任务都要上 Agent？"
    **答案：**

    如果任务流程稳定、规则明确、一步即可完成，就用普通 workflow 或后端逻辑更可靠。Agent 更适合信息不完整、需要动态选工具、多步推理和失败反馈的任务。
