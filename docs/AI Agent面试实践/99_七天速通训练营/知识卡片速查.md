# 知识卡片速查

> 面试前一天只看这个文件。每个知识点按“一句话定义 → 30 秒回答 → 项目举证”记忆。

---

## 1. Agent

**一句话定义：**Agent 是能基于目标、上下文和工具，在多步任务中自主决策并执行动作的系统。

**30 秒回答：**普通聊天机器人主要生成文本，Agent 还会调用工具、读取状态、观察结果并继续行动。一个可落地 Agent 需要模型、上下文、工具、编排和治理。

**项目举证：**工具型 Agent 会检索证据、选择工具、执行动作、处理失败并返回可追踪结果。

---

## 2. RAG

**一句话定义：**RAG 是检索增强生成，让模型回答前先检索外部知识。

**30 秒回答：**RAG 分为索引、检索、增强、生成四步，用外部知识减少幻觉并提升可追溯性。生产中要关注 Chunk、Embedding、TopK、Rerank 和引用。

**项目举证：**知识库 Agent 会召回文档片段、返回引用并对回答忠实度做评测。

---

## 3. Tool Calling

**一句话定义：**模型选择工具和参数，应用侧执行工具并回填结果。

**30 秒回答：**Tool Calling 的关键边界是模型不直接执行外部动作，系统负责执行和安全控制。工具 schema 要命名清晰、参数少而准、返回结构化。

**项目举证：**业务 API 工具要负责参数校验、权限检查和错误返回。

---

## 4. LangGraph

**一句话定义：**LangGraph 是用 State、Node、Edge 编排复杂 Agent 工作流的图框架。

**30 秒回答：**它适合有状态、条件分支和循环重试的任务。State 保存共享数据，Node 处理步骤，Edge 决定流转，Conditional Edge 根据状态路由。

**项目举证：**工具执行失败可以从 `tool_execute` 回到 `tool_select` 做受控重试。

---

## 5. Eval Harness

**一句话定义：**Eval Harness 是固定评测集、指标统计和自动报告组成的 Agent 回归测试系统。

**30 秒回答：**Agent 输出不稳定，所以不能只靠手动 Demo。Eval Harness 用固定 Case 衡量 Pass@1、Recall、延迟、自愈率等指标，证明优化是否有效。

**项目举证：**Agent 可维护固定评测 Case，并输出 Markdown / JSON 报告做回归。

---

## 6. 工具安全

**一句话定义：**大模型触发外部动作必须经过执行层安全校验。

**30 秒回答：**Prompt 不是安全边界，执行层要做工具白名单、参数校验、权限分级、审计和高风险动作审批。

**项目举证：**高风险工具应返回实际执行动作、审批状态和审计记录。

---

## 7. 上下文工程

**一句话定义：**上下文工程是管理模型当前能看到什么信息。

**30 秒回答：**它包含 Prompt、历史消息、RAG 结果、工具返回、摘要压缩和状态注入。很多 Agent 失败不是模型差，而是上下文错、脏、乱或太长。

**项目举证：**多轮 Agent 可通过 conversation_id 注入近期历史并按需检索长期记忆。

---

## 8. MCP

**一句话定义：**MCP 是连接 AI 应用与外部工具/数据源的标准协议。

**30 秒回答：**它把工具接入从每个系统单独适配，变成统一的 Client-Server 协议，像 AI 世界的 USB-C。

**项目举证：**搜索、文件与业务 API 都可以通过 MCP 服务暴露给 Agent。

---

## 9. Planner

**一句话定义：**Planner 把复杂任务拆成可执行子任务。

**30 秒回答：**简单问题可以直接执行，复杂分析需要先生成 Query Plan，再逐步查询和汇总。Planner 不一定是复杂多 Agent，可以是轻量结构化节点。

**项目举证：**Planner 可把复杂调研任务拆成检索、比对、校验和总结步骤。

---

## 10. Skill

**一句话定义：**Skill 是完成一类任务的可复用行为流程。

**30 秒回答：**Tool 解决能做什么，Skill 解决如何稳定做好。Skill 通常包含触发条件、工作流、Prompt、工具集合、约束、示例和评估。

**项目举证：**知识库问答、报告生成和代码修复都可以沉淀为不同 Skill。
