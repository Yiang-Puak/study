## 10 分钟知识卡片

### 1. Tool Calling 闭环

```text
用户问题
  ↓
模型判断是否需要工具
  ↓
模型输出工具名 + 参数
  ↓
应用侧执行工具
  ↓
工具结果回填给模型
  ↓
模型继续推理或输出最终答案
```

### 2. 关键边界

模型不直接执行工具，模型只负责选择工具和生成参数；真正执行发生在应用侧。

### 3. 好的 Tool Schema

- 名称清晰；
- 参数少而准；
- 描述包含适用场景；
- 返回结果结构化；
- 高风险工具需要权限和审批。

---

## 项目举证

工具型 Agent 中，业务查询 API 就是典型工具：

- LLM 选择工具并生成参数；
- 后端校验参数与权限；
- 工具执行真实动作；
- 结果回填给解释节点；
- 失败时错误信息回到编排层决定重试或停止。

## 面试训练

??? question "Q1：Tool Calling 是什么？"
    Tool Calling 是让模型以结构化方式选择外部工具并生成参数，应用程序负责执行工具，再把工具结果回填给模型。它让 LLM 从文本生成器变成可以连接数据库、API、搜索、文件系统的任务执行器。

??? question "Q2：Function Calling 和 Tool Calling 有什么区别？"
    Function Calling 更强调模型输出函数名和参数；Tool Calling 是更广义的工具调用机制，工具可以是函数、API、数据库、搜索、文件系统或 MCP 服务。工程上二者常常接近。

??? question "Q3：MCP 是什么？"
    MCP 是 Model Context Protocol，用统一协议把外部工具和数据源接入 AI 应用。它像 AI 世界的 USB-C，把每个系统都单独集成，变成统一的 Client-Server 工具协议。

??? question "Q4：工具调用失败怎么办？"
    要有超时、重试、错误结构化返回、降级策略和人工确认。失败信息要回到编排层，决定重试、换工具还是停止。
