## 10 分钟知识卡片

### 1. RAG 四阶段

```text
索引：文档/知识对象 → Chunk → Embedding → 向量库
检索：Query → Embedding → TopK 召回
增强：把召回内容注入 Prompt
生成：LLM 基于上下文回答或生成结构化结果
```

### 2. RAG 的核心价值

- 减少模型幻觉；
- 接入实时/私有知识；
- 让答案可溯源；
- 降低把所有知识塞进 Prompt 的成本。

### 3. 结构化检索

结构化检索不一定召回长文档，也可能召回：

- 元数据；
- 字段说明；
- 规则定义；
- 工具说明。

## 项目举证

结构化检索示例：


## 面试训练

??? question "Q1：RAG 的完整流程是什么？"
    RAG 分为索引、检索、增强、生成四步。索引阶段把文档切成 chunk 并向量化；检索阶段用用户 query 召回相关 chunk；增强阶段把召回内容拼入 Prompt；生成阶段由 LLM 基于上下文回答。生产中还会加混合检索、Reranker、引用和评测。

??? question "Q2：Chunk 太大或太小有什么问题？"
    Chunk 太大，噪声多，召回不精准；Chunk 太小，语义不完整，模型缺少上下文。一般要结合文档结构、标题、段落和任务类型调参。

??? question "Q3：结构化检索和文档 RAG 有什么区别？"
    文档 RAG 多召回自然语言片段，结构化检索可能召回字段、规则、工具说明或元数据。前者更关注引用和回答忠实度，后者还要关注结构约束与下游校验。

??? question "Q4：怎么评估 RAG 效果？"
    检索阶段看 Hit@K、Recall@K、MRR、NDCG；生成阶段看答案正确率、忠实度、引用准确率。
