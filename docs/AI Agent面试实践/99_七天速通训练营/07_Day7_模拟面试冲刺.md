## 高频题 15 连

??? question "1. Agent 和普通聊天机器人有什么区别？"
    Agent 不只生成文本，还能基于目标、上下文和工具执行动作。它需要状态、工具、编排、观测和安全。

??? question "2. RAG 的完整流程是什么？"
    索引、检索、增强、生成。索引阶段切块并向量化；检索阶段召回相关 chunk；增强阶段注入 Prompt；生成阶段基于上下文回答。

??? question "3. Tool Calling 是什么？"
    模型输出工具名和参数，应用侧执行工具，再把结果回填给模型。模型不直接执行工具，真正执行和安全控制在应用侧。

??? question "4. LangGraph 为什么适合 Agent 工作流？"
    因为它有显式 State、Node、Edge 和条件边，适合状态流转、分支和循环。

??? question "5. 文档 RAG 和结构化检索区别是什么？"
    文档 RAG 主要检索自然语言片段，结构化检索可能召回字段、规则、工具说明或元数据。后者更强调结构约束和下游校验。

??? question "6. 怎么防止模型调用危险工具？"
    不能只靠 Prompt。执行层要做白名单、参数校验、权限控制和审计，高风险动作要加人审。

??? question "7. Eval Harness 为什么重要？"
    因为 Agent 输出不稳定，不能只靠 Demo。Eval Harness 用固定 Case 和指标做回归，证明优化是否真的有效。

??? question "8. Pass@1 是什么？"
    Pass@1 是首次生成并完成任务的比例，反映一次尝试就得到正确结果的能力。

??? question "9. Self-Correction Rate 是什么？"
    发生重试的 Case 中最终修复成功的比例，衡量 Agent 从失败中恢复的能力。

??? question "10. 什么是上下文工程？"
    上下文工程不是写一个 Prompt，而是持续决定模型当前应该看到什么，包括历史、RAG 结果、工具返回、压缩摘要和任务状态。

??? question "11. MCP 是什么？"
    MCP 是把工具和数据源标准化接入 AI 应用的协议，类似 AI 世界的 USB-C，减少每个系统都单独集成的成本。

??? question "12. Skill 和 Tool 有什么区别？"
    Tool 是单个能力接口，Skill 是完成一类任务的可复用流程，包含触发条件、工作流、Prompt、工具集合、约束和评估。

??? question "13. 什么时候需要 Planner？"
    当问题需要多步骤、多子查询或中间结果汇总时，需要 Planner 拆成 Query Plan。简单问题直接走标准链路即可。

??? question "14. 你怎么做 Agent 可观测性？"
    记录每次运行的 intent、retrieved context、selected tool、tool args、tool result、token、latency、retry_count 和 execution_trace，用于定位失败节点。

??? question "15. 一个 Agent 项目怎样体现工程完整性？"
    不只展示模型回答，还要说明上下文来源、工具边界、失败恢复、可观测链路、安全护栏和评测指标。
