# Workflow 与 LangGraph 真题与工程追问

## 一、示范真题

### 题 1：一个 Agent 任务既有固定审批，又有动态工具选择，你怎么编排

**回答骨架：**

1. 把固定合规步骤做成 Workflow 节点。
2. 在需要查证据或选工具的局部节点放 Agent 决策。
3. 用 State 存目标、证据、工具结果、审批状态。
4. 高风险边上加入人工中断，失败路径写清重试和退出条件。

### 题 2：LangGraph 为什么比单条 Chain 更适合复杂任务

**先答结论：**复杂任务需要显式状态、条件分支、循环、恢复和观测，单条线性 Chain 很难把这些边界表达清楚。

**继续展开：**

- 条件边可以按审核分数、工具错误或证据质量路由。
- 循环边可以表达修改、重试和反思，但要有上限。
- Checkpoint 和中断适合长任务与人工确认。

### 题 3：什么情况下你不会做 Multi-Agent

**回答骨架：**

1. 任务本身线性，单 Agent 能稳定完成。
2. 角色拆分后共享上下文仍高度耦合。
3. 额外通信和评测成本超过收益。
4. 先用单 Agent 加工具和清晰 Workflow 建基线，再用 Eval 比较拆分收益。

## 二、工程追问清单

| 追问 | 回答提示 |
| :--- | :--- |
| Graph 循环怎么防死循环 | 最大步数、退出条件、失败策略 |
| State 太大怎么办 | 存引用、摘要、分离大对象 |
| 多节点怎么做 Trace | 每个节点记录输入摘要、输出和决策 |
| 人工确认怎么接入 | interrupt、审批状态、恢复执行 |
| 多 Agent 结果冲突怎么办 | 汇总策略、裁决节点、证据约束 |
