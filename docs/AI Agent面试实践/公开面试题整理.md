# Agent 公开面试题整理

> 本页收集公开题目方向，用于训练题感。题目需要回到专题页校准原理，不把零散面经当成唯一知识来源。

## 一、来源分级

| 等级 | 来源 | 使用方式 |
| :--- | :--- | :--- |
| A | 官方文档、官方技术博客、岗位描述 | 判断能力重点和标准术语 |
| B | 可追溯的公开面经、公开分享 | 整理题型与追问 |
| C | 社区题单与二次汇总 | 去重后作为补充练习 |
| D | 无法核验的转述 | 只保留方向，不写成必考结论 |

## 二、Agent 与编排题

| 题目 | 主要考察 | 推荐回补 |
| :--- | :--- | :--- |
| Agent 与普通聊天系统的区别是什么？ | 能力边界、工具、状态 | Agent 基础 |
| 固定 Workflow 和 Agent 怎么选？ | 可控性、复杂度、成本 | Agent 编排 |
| 为什么先做单 Agent，再考虑 Multi-Agent？ | 复杂度和失败面 | Multi-Agent |
| LangGraph 为什么适合有循环和人工确认的流程？ | State、条件边、持久化 | LangGraph |
| Workflow 和 Agent 什么时候混用？ | 固定流程、开放决策 | Workflow |

## 三、RAG 与 Context 题

| 题目 | 主要考察 | 推荐回补 |
| :--- | :--- | :--- |
| RAG 的完整链路怎么画？ | 索引、召回、增强、生成 | RAG 总览 |
| Chunk 太大或太小会发生什么？ | 检索粒度、上下文完整性 | Chunk |
| Hybrid Retrieval 和 Rerank 分别解决什么问题？ | 召回与精排 | 检索优化 |
| 长上下文模型出现后还需要 RAG 吗？ | 成本、时效、引用、检索 | Context 与 RAG |
| 怎么判断一条回答是否有依据？ | Citation、groundedness、拒答 | RAG Eval |

## 四、Tool、MCP 与安全题

| 题目 | 主要考察 | 推荐回补 |
| :--- | :--- | :--- |
| Tool Calling 的执行权在模型还是应用侧？ | 工具边界 | Tool 设计 |
| 工具参数为什么要做 schema 和校验？ | 结构化接口 | Tool 设计 |
| MCP 的 Host、Client、Server 各自做什么？ | 协议结构 | MCP |
| Prompt Injection 对工具型 Agent 有什么风险？ | 权限与隔离 | Safety |
| 高风险工具为什么要加人工确认？ | HITL、审计 | Safety |

## 五、Eval 与项目追问题

| 题目 | 主要考察 | 推荐回补 |
| :--- | :--- | :--- |
| Agent 评测集怎么构造？ | case、指标、回归 | Eval |
| 一次失败任务如何用 Trace 排查？ | 调用链和错误分类 | Trace |
| 工具型 Agent 怎么限制危险操作？ | 执行层安全 | Safety |
| 检索型 Agent 怎么减少错误上下文注入？ | 召回、过滤、引用 | RAG |
| 自愈重试如何避免无限循环？ | 退出条件和失败策略 | Reliability |
