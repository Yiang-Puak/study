# 前沿与趋势

> 这部分用来准备面试里的开放题：为什么现在大家重新讨论 Agent？你怎么看 Agentic RAG、多模态、成本和安全？

## 2026 前后高频话题

| 话题 | 面试会怎么问 | 回答抓手 |
| :--- | :--- | :--- |
| Agentic RAG | 传统 RAG 和 Agentic RAG 区别？ | 检索不再是单次链路，而是可规划、可迭代、可用工具修正 |
| MCP | MCP 解决了什么问题？ | 工具上下文协议化，降低模型应用接工具的碎片化成本 |
| 多 Agent | 多 Agent 什么时候值得用？ | 复杂任务拆分、角色专业化，但要付出通信和一致性成本 |
| 多模态 Agent | 文档、图片、表格怎么进 Agent？ | 解析、结构化、证据定位和跨模态引用 |
| 成本优化 | LLM 应用怎么降成本？ | 小模型路由、缓存、检索裁剪、批处理、prompt 压缩 |
| 安全治理 | Agent 最大风险是什么？ | 工具越权、Prompt 注入、数据泄露、不可审计行为 |

## 项目联想

| 趋势 | 可以怎样改自己的项目 |
| :--- | :--- |
| Agentic RAG | 给 RAG 增加 query rewrite、multi-hop retrieval、answer verification |
| MCP | 把文件检索、数据库查询、网页搜索封装成统一工具接口 |
| Trace | 记录每个节点的输入、输出、耗时、token 和错误 |
| 成本优化 | 为简单问题走规则或小模型，复杂问题再走强模型 |
| HITL | 高风险 SQL、删除类工具、外部发送动作必须人工确认 |
