## P1：拉开差距的能力

| 能力 | 面试官常问法 | 训练方式 |
| :--- | :--- | :--- |
| Eval / Trace | 你怎么知道 Agent 变好了？ | 用 case 集、trace、延迟、成本和失败分类回答 |
| Context 工程 | 长上下文、记忆和检索怎么取舍？ | 画出短期消息、摘要、长期记忆、检索上下文边界 |
| Prompt 工程 | 为什么不是简单写提示词？ | 用结构化输出、few-shot、约束和回归测试回答 |
| 安全边界 | 工具误调用、SQL 注入、Prompt 注入怎么办？ | 用 schema 校验、权限隔离、HITL、审计日志回答 |
| 部署质量 | 项目怎么上线、怎么监控？ | 用 CI、Docker、日志、健康检查和回滚策略回答 |

## P2：短期可以后补

| 能力 | 什么时候学 |
| :--- | :--- |
| 微调 / LoRA | 岗位 JD 明确写模型训练或领域适配时 |
| 多模态 Agent | 岗位涉及 OCR、图像理解、文档解析时 |
| 高并发推理服务 | 岗位偏平台、网关、模型服务时 |
| Kubernetes | 岗位偏基础设施或企业平台时 |

## 自检

刷题、看项目或模拟面试后，记录三件事：

```text
今天能稳定讲清的 1 个知识点：
今天暴露的 1 个薄弱点：
明天要回流到哪一页：
```

把答不稳的问题写进 [薄弱点回流表](薄弱点回流表.md)，不要只收藏链接。
