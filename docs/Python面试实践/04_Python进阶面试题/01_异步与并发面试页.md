# Python 异步与并发面试页

> Agent 工程里大量工作是 I/O：模型请求、工具 API、检索服务和流式输出。面试时要把线程、进程和异步的边界讲清。

## 一句话定义

并发关注多个任务如何交替推进；异步是 Python 处理大量等待型 I/O 的一种协作式并发方式。

## 核心对比

| 方案 | 适合 | 不适合 |
| :--- | :--- | :--- |
| `threading` | 阻塞 I/O、接旧同步库 | CPU 密集 Python 字节码 |
| `asyncio` | 大量可 await I/O、服务端并发 | 把阻塞函数直接塞进事件循环 |
| `multiprocessing` | CPU 密集计算 | 频繁共享大对象 |

| task | 被调度执行的协程对象 |

## 高频真题

### Q1：GIL 下多线程还有用吗

**答题要点：**

1. 对 CPU 密集 Python 代码，线程受 GIL 限制。
2. 对 I/O 密集任务，线程仍有价值，因为等待 I/O 时可以切换。
3. 大量网络 I/O 可考虑 `asyncio`，CPU 密集可考虑多进程或原生扩展。

### Q2：异步代码为什么也会卡住

**答题要点：**

- 在事件循环里调用阻塞函数。
- 协程内部长时间 CPU 计算不让出控制权。
- 外部库并没有异步接口。

| 场景 | Python 选择 |
| :--- | :--- |
| 多工具 API 并发等待 | `asyncio` 或异步客户端 |
