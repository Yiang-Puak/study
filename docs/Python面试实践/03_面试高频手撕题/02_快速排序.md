"""
【面试高频】快速排序 —— 分治法 + 原地分区

题目：实现快速排序，并分析时间复杂度和稳定性。

要求：
- 原地排序（不额外分配大量空间）
- 能处理重复元素
- 能说清时间/空间复杂度和稳定性
"""

import random


def quick_sort(arr: list, left: int = 0, right: int = None) -> None:
    """
    原地快速排序 —— 面试标准写法

    思路：
    1. 选择基准值 pivot（通常选最右或随机）
    2. 分区：将小于 pivot 的放左边，大于 pivot 的放右边
    3. 递归排序左右两部分
    """
    if right is None:
        right = len(arr) - 1

    if left >= right:
        return

    # 随机选 pivot 并交换到末尾，避免最坏情况
    pivot_idx = random.randint(left, right)
    arr[pivot_idx], arr[right] = arr[right], arr[pivot_idx]

    # 分区操作
    pivot = arr[right]
    store_idx = left  # 小于 pivot 的区域边界

    for i in range(left, right):
        if arr[i] < pivot:
            arr[store_idx], arr[i] = arr[i], arr[store_idx]
            store_idx += 1

    # 将 pivot 放到正确位置
    arr[store_idx], arr[right] = arr[right], arr[store_idx]

    # 递归排序左右两部分
    quick_sort(arr, left, store_idx - 1)
    quick_sort(arr, store_idx + 1, right)


def quick_sort_three_way(arr: list, left: int = 0, right: int = None) -> None:
    """
    三路快排 —— 处理大量重复元素时的优化版本

    面试加分项！将数组分为三部分：
    - 小于 pivot
    - 等于 pivot
    - 大于 pivot

    对于重复元素多的场景（如 [1,1,1,2,2,2]），时间复杂度从 O(n²) 降到 O(n)
    """
    if right is None:
        right = len(arr) - 1

    if left >= right:
        return

    # 随机选 pivot
    pivot_idx = random.randint(left, right)
    arr[pivot_idx], arr[left] = arr[left], arr[pivot_idx]
    pivot = arr[left]

    # 三路分区
    lt = left      # 小于 pivot 的右边界（arr[left..lt-1] < pivot）
    gt = right     # 大于 pivot 的左边界（arr[gt+1..right] > pivot）
    i = left + 1   # 当前扫描位置

    while i <= gt:
        if arr[i] < pivot:
            arr[lt], arr[i] = arr[i], arr[lt]
            lt += 1
            i += 1
        elif arr[i] > pivot:
            arr[i], arr[gt] = arr[gt], arr[i]
            gt -= 1
        else:  # arr[i] == pivot
            i += 1

    # 递归排序小于和大于的部分（等于的部分已经有序）
    quick_sort_three_way(arr, left, lt - 1)
    quick_sort_three_way(arr, gt + 1, right)

"""
【面试追问】

Q: 时间复杂度？
A: 平均 O(n log n)，最坏 O(n²)（已排序数组 + 固定选最左 pivot）。
   随机选 pivot 可将最坏情况概率降到极低。

Q: 空间复杂度？
A: 原地排序 O(log n) —— 递归栈深度。
   最坏 O(n)（已排序时递归深度为 n）。

Q: 快排是稳定的吗？
A: 不是！分区时会交换元素，相等元素的相对顺序可能改变。
   例如 [3a, 2, 3b]，分区后 3a 和 3b 可能交换位置。
   需要稳定排序时用归并排序。

Q: 为什么比归并排序快？
A: 1）原地排序，不需要额外数组；2）cache 局部性更好。
   但实际工程中 Python 的 `sorted()` 是 Timsort（归并+插入），通常更快。

Q: 三路快排什么时候用？
A: 数据中有大量重复元素时（如分类标签、状态码），
   传统快排会退化，三路快排能达到 O(n)。
