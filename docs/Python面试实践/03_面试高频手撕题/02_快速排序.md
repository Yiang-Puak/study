## 解题思路

快速排序把数组按 pivot 分成两边：

1. 选一个基准值。
2. 把小于基准的元素交换到左侧。
3. 把 pivot 放回最终位置。
4. 递归处理左右区间。

面试时先把“区间缩小”讲清楚，再补随机 pivot 和三路快排。

## 参考实现

```python
import random

def quick_sort(arr: list[int], left: int = 0, right: int | None = None) -> None:
    if right is None:
        right = len(arr) - 1
    if left >= right:
        return

    pivot_index = random.randint(left, right)
    arr[pivot_index], arr[right] = arr[right], arr[pivot_index]
    pivot = arr[right]

    store = left
    for index in range(left, right):
        if arr[index] < pivot:
            arr[store], arr[index] = arr[index], arr[store]
            store += 1

    arr[store], arr[right] = arr[right], arr[store]
    quick_sort(arr, left, store - 1)
    quick_sort(arr, store + 1, right)
```

## 复杂度

| 指标 | 结论 |
| :--- | :--- |
| 平均时间 | O(n log n) |
| 最坏时间 | O(n^2)，分区极不均衡时出现 |
| 额外空间 | 平均 O(log n)，来自递归栈 |
| 稳定性 | 不稳定，交换会打乱相等元素相对顺序 |

## 易错点

1. `left >= right` 是递归终止条件，空区间和单元素区间都要停。
2. partition 后 pivot 已经就位，递归区间不要再包含它。
3. 数据里重复元素很多时，普通二路分区可能退化，三路快排更稳。

## 面试追问

| 追问 | 回答抓手 |
| :--- | :--- |
| 为什么随机 pivot | 降低连续不均衡分区的概率 |
| 和归并排序怎么比 | 快排通常原地，归并更稳定但要额外空间 |
| Python 工程里直接用什么 | 优先用内置排序，手写快排主要用于考察分治和边界 |
