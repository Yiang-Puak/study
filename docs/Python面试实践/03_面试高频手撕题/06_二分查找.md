# 二分查找

## 解题思路

本页采用闭区间 `[left, right]`：

1. 初始 `left = 0`，`right = len(nums) - 1`。
2. 当 `left <= right` 时，当前区间仍有元素。
3. 比较中点后，搜索区间必须严格缩小。

区间定义稳定，后续“第一个大于”“最后一个等于”这类变体才不容易写乱。

## 参考实现

```python
def binary_search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        if nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
```

## 复杂度

| 指标 | 结论 |
| :--- | :--- |
| 时间 | O(log n) |
| 空间 | 迭代版 O(1) |

## 易错点

1. 无序数组不能直接二分。
2. `left <= right` 与 `right = mid - 1` 要和闭区间定义配套。
3. 有重复元素时，普通二分只保证返回一个命中位置，不保证第一个或最后一个。

## 面试追问

| 追问 | 回答抓手 |
| :--- | :--- |
| 为什么是 O(log n) | 每轮把候选区间折半 |
| 怎么找第一个等于 | 命中后记录答案，继续向左收缩 |
| 怎么找第一个大于 | 条件满足时记录位置，右边界继续左移 |
