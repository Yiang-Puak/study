## 这个知识点解决什么问题

当一段逻辑要被复用、测试或组合时，就应该抽成函数。学会函数后，后面的装饰器、递归、工具封装和手撕题都更容易读。

## 必会概念

| 概念 | 先记住 |
| :--- | :--- |
| 定义与调用 | 用 `def` 定义，用函数名加括号调用 |
| 参数 | 调用者交给函数的输入 |
| 返回值 | 函数处理后的结果，没写 `return` 时返回 `None` |
| 作用域 | 函数内部变量默认只在函数内可见 |
| 可变参数 | `*args` 收位置参数，`**kwargs` 收关键字参数 |

## 最小示例

```python
def add(a, b):
    return a + b

def sum_all(*numbers):
    return sum(numbers)

print(add(3, 5))
print(sum_all(1, 2, 3))
```

## 参数和返回值

位置参数按顺序传入，关键字参数按名字传入：

```python
def create_user(name, age, city):
    return {"name": name, "age": age, "city": city}

create_user("张三", 20, "北京")
create_user(age=20, city="北京", name="张三")
```

`print()` 只负责显示；`return` 才把结果交回调用者继续计算。

## 面试高频坑：可变默认参数

```python
def add_item(item, items=[]):
    items.append(item)
    return items
```

默认参数在函数定义时创建一次，上面的列表会在多次调用之间共享。更稳的写法是：

```python
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

## 容易混淆的点

1. 函数内部的局部变量默认不会泄漏到函数外。
2. `*args` 在函数内部是元组，`**kwargs` 是字典。
3. `global` 能改全局变量，但业务代码里应优先把输入和输出写清楚。

## 常见考法

1. 解释 `return` 和 `print` 的区别。
2. 写出可变默认参数的错误示例与修复写法。
3. 说明 `*args` 与 `**kwargs` 的用途。
