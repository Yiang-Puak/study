# 异常与上下文管理面试页

> Python 面试不只看“会不会 try except”，更看你能不能设计失败边界和资源释放。

## 一、异常处理

### 一句话定义

异常是把非正常控制流显式交给调用方处理的机制。

## 二、异常设计原则

| 原则 | 说明 |
| :--- | :--- |
| 捕获范围要窄 | 不用裸 `except` 吞掉真实错误 |
| 失败要可解释 | 错误信息带上下文 |
| 边界层再翻译 | 底层异常可在接口层转成稳定回执 |
| 需要时定义自定义异常 | 区分参数错、依赖错、权限错 |

## 三、`with` 在做什么

上下文管理器把资源进入和退出封装起来，确保正常执行和异常路径都能释放资源。

```python
with open("notes.md", "r", encoding="utf-8") as file:
    content = file.read()
```

等价心智模型：

```text
__enter__ -> 使用资源 -> __exit__
```

## 四、高频真题

### Q1：`finally` 一定会执行吗

**答题要点：**通常用于保证清理逻辑执行，但不要在 `finally` 里随意覆盖返回值或吞掉异常。

### Q2：为什么文件、锁、连接推荐用 `with`

**答题要点：**资源释放不依赖每条路径手写 close，异常时也能统一收口。

### Q3：Tool 调用失败时 Python 异常怎么映射

**答题要点：**

1. 参数错误和依赖错误分开。
2. 记录原始错误上下文。
3. 对 Agent 返回结构化失败回执，避免模型猜。

## 五、Agent 工程落点

| Python 机制 | Agent 场景 |
| :--- | :--- |
| 自定义异常 | 工具失败分类 |
| 上下文管理器 | 文件、连接、临时资源 |
| `finally` | 审计、清理和释放 |
| 错误回执 | 让模型看到可恢复观察 |
