## 先看选择表

| 结构 | 典型用途 | 是否可变 | 查找特征 |
| :--- | :--- | :--- | :--- |
| `list` | 保留顺序的一组元素 | 是 | 索引访问快，成员查找通常要遍历 |
| `tuple` | 不希望替换元素的记录 | 否 | 支持索引和切片 |
| `dict` | 用 key 找 value | 是 | 平均 O(1) |
| `set` | 去重、集合运算、成员判断 | 是 | 平均 O(1) |

## `list`：按顺序放元素

```python
scores = [80, 90, 95]
scores.append(100)
scores[0] = 88
print(scores[1:])  # [90, 95, 100]
```

`list` 底层可把它先理解成动态数组：

- `scores[i]` 是 O(1)。
- 末尾 `append()` 均摊 O(1)。
- 中间插入和删除通常是 O(n)，因为后面的元素要移动。

## `dict`：按 key 找 value

```python
phone_book = {"张三": "13800000001", "李四": "13800000002"}
phone_book["王五"] = "13800000003"
print(phone_book.get("赵六", "未找到"))
```

`dict` 底层核心是哈希表。key 的哈希值要稳定，因此列表这类可变对象不能直接做 key。

面试里要顺带说清：

- 平均查找、插入、删除都是 O(1)。
- 哈希冲突和扩容会影响常数与极端情况。
- Python 3.7+ 的 `dict` 保持插入顺序，但它首先仍然是映射结构。

## `tuple`：记录一组不想替换的值

```python
point = (116.4, 39.9)
x, y = point
single = (42,)
```

`tuple` 自身不可变，因此适合返回多个结果、表达固定记录，满足条件时也能作为字典 key。

容易被追问的坑是：元组里如果装着列表，列表内部仍可修改。不可变约束的是元组自身的元素引用。

## `set`：去重和集合判断

```python
skills = {"rag", "tool", "rag"}
print(skills)  # 自动去重

left = {"rag", "eval"}
right = {"eval", "safety"}
print(left & right)  # 交集
```

需要快速判断“某元素在不在一批值里”时，`set` 往往比 `list` 更合适。

## 容易混淆的点

1. `dict` 的 `{}` 是空字典，空集合要写 `set()`。
2. `tuple` 不可变不等于“里面所有对象都永远不变”。
3. `set` 适合成员判断和去重，但不要依赖它的显示顺序表达业务语义。

## 常见考法

1. 为什么 `list.append()` 和 `list.insert()` 的复杂度不同。
2. 为什么 `dict` 查找通常比 `list` 成员判断快。
3. 什么时候选择 `tuple` 而不是 `list`。
4. 如何求两个列表的交集并说明复杂度。

## 小练习

1. 用 `dict` 统计一段字符串里每个字符出现的次数。
2. 用 `set` 找两个列表共有的元素。
3. 把一组学生成绩放进 `list`，找出最后三个成绩。
