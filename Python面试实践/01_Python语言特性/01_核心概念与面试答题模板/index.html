    01 | Python 语言特性 —— 核心概念与面试答题模板¶

    一、核心概念速记¶

    1. 可变对象 vs 不可变对象（⭐⭐⭐⭐）¶

    类型 示例 特点 面试陷阱
    不可变 str, int, float, tuple, bool 值变则对象 ID 变 def func(items=[]) 默认参数共享同一列表
    可变 list, dict, set 原地修改，ID 不变 a = [1,2]; b = a; b.append(3) → a 也被改

    面试必考陷阱： 

    def add_item(item, cache=[]):
    cache.append(item)
    return cache

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2]  ← 坑！cache 被共享了
    解决：def add_item(item, cache=None): cache = cache or []

    2. 装饰器与闭包（⭐⭐⭐⭐）¶

    • 本质：接收函数、返回函数的高阶函数
    • 闭包：内层函数引用外层函数的变量，外层函数返回内层函数
    • 带参数装饰器：需要三层嵌套（装饰器参数 → 装饰器函数 → 被装饰函数）
    • @functools.wraps：保留被装饰函数的 __name__、__doc__
    import functools

def my_decorator(func):
    @functools.wraps(func)  # 面试必写！
    def wrapper(*args, **kwargs):
        print("before")
        result = func(*args, **kwargs)
        print("after")
        return result
    return wrapper

    3. 生成器与迭代器（⭐⭐⭐⭐）¶

    概念 定义 面试关键词
    迭代器 实现 __iter__ + __next__ iter(), next(), StopIteration
    生成器 含 yield 的函数 惰性求值、节省内存
    yield from 委托迭代，避免手动循环 Python 3.3+
    .send() 向生成器发送值 协程基础

    面试金句： - 列表推导式 [x for x in range(10)] → 内存中存完整列表 - 生成器表达式 (x for x in range(10)) → 惰性计算，几乎不耗内存

    4. GIL 与并发（⭐⭐⭐⭐）¶

    • GIL（Global Interpreter Lock）：同一时刻只有一个线程执行 Python 字节码
    • 影响：多线程对 CPU 密集型任务无效，甚至因为上下文切换更慢
    • 正确用法：
    • I/O 密集型（网络请求/文件读写）：threading 或 asyncio
    • CPU 密集型（计算/数据处理）：multiprocessing

    5. 浅拷贝 vs 深拷贝（⭐⭐⭐）¶

    方法 行为 面试关键词
    copy.copy() 只拷贝最外层容器，内部对象共享引用 浅拷贝
    copy.deepcopy() 递归拷贝所有层级，完全独立 深拷贝 
    import copy
a = [[1, 2], [3, 4]]
b = copy.copy(a)      # 浅拷贝
b[0].append(5)        # a[0] 也会变成 [1,2,5]！

c = copy.deepcopy(a)  # 深拷贝
c[0].append(6)        # a[0] 不变

    二、面试真题 + 标准答案¶

    【真题 1】Python 的可变对象和不可变对象有什么区别？举例子。（⭐⭐⭐⭐）¶

    答题模板： 1. 核心定义：不可变对象创建后内容不可修改（str, int, tuple），任何"修改"操作都会创建新对象；可变对象可以在原地修改（list, dict, set），修改后对象 ID 不变。 2. 底层原理：不可变对象便于作为字典 key 和集合元素（需要 hash 值稳定）；可变对象支持原地修改，性能更好但线程不安全。 3. 面试陷阱：tuple 本身不可变，但如果内部有可变对象（如 ([1,2],)），内部列表仍然可以修改。 4. 常见坑点：可变对象做函数默认参数导致状态残留；浅拷贝嵌套结构时内部对象共享引用。 5. 生产优化：多线程环境下用不可变数据结构（如 frozenset）避免锁竞争；需要共享状态时优先考虑 deepcopy。

    【真题 2】装饰器的原理是什么？@functools.wraps 有什么用？（⭐⭐⭐⭐）¶

    答题模板： 1. 核心定义：装饰器是接收函数作为参数、返回新函数的高阶函数，用于在不修改原函数代码的情况下增强功能。 2. 实现原理：@decorator 等价于 func = decorator(func)，返回的 wrapper 函数包裹了原函数的执行逻辑。 3. 三层嵌套：带参数的装饰器需要三层函数（参数层 → 装饰器层 → wrapper 层）。 4. @functools.wraps：如果不加，wrapper 函数会覆盖原函数的 __name__、__doc__ 等元信息，导致日志和调试困难。wraps(func) 内部调用 update_wrapper，将原函数的元信息复制到 wrapper 上。 5. 生产优化：装饰器逻辑尽量轻量，避免在装饰器里做耗时操作；记录装饰器调用日志便于排查问题；类装饰器用 __call__ 实现。

    【真题 3】生成器和迭代器的区别？yield from 有什么用？（⭐⭐⭐⭐）¶

    答题模板： 1. 核心定义：迭代器是实现了 __iter__ 和 __next__ 的对象；生成器是含有 yield 关键字、自动实现迭代器协议的函数。 2. 本质区别：迭代器需要手动实现两个方法；生成器由 Python 自动包装为迭代器，写法更简洁。 3. yield from：Python 3.3 引入，用于委托迭代。例如 yield from sub_generator() 等价于手动遍历 sub_generator 并逐个 yield，代码更简洁，还能正确传递 .send() 和 .throw()。 4. 常见坑点：生成器只能遍历一次，第二次遍历时为空；生成器表达式比列表推导式省内存但只能顺序访问；yield 在协程中的使用需要理解状态机。 5. 生产优化：处理大数据集时用生成器逐行读取（如日志分析）；itertools 模块提供高效的生成器工具函数；管道式处理数据避免全部加载到内存。

    【真题 4】GIL 是什么？多线程为什么不能加速 CPU 密集型任务？（⭐⭐⭐⭐）¶

    答题模板： 1. 核心定义：GIL（Global Interpreter Lock）是 CPython 解释器的全局锁，保证同一时刻只有一个线程在执行 Python 字节码。 2. 为什么存在：简化内存管理（引用计数不需要线程锁），避免多线程竞争导致的死锁和性能下降。 3. 多线程无效的原因：CPU 密集型任务需要持续占用 CPU，GIL 让多线程实际上串行执行，加上线程切换开销，可能比单线程更慢。 4. 正确方案：I/O 密集型用 threading/asyncio（等待 I/O 时释放 GIL）；CPU 密集型用 multiprocessing（每个进程独立 GIL）或改用 C 扩展/numba/Cython。 5. 生产优化：concurrent.futures.ThreadPoolExecutor 管理线程池；ProcessPoolExecutor 管理进程池；异步框架（FastAPI/Tornado）处理高并发 I/O。

    【真题 5】深浅拷贝的区别？什么场景下必须用深拷贝？（⭐⭐⭐）¶

    答题模板： 1. 核心定义：浅拷贝创建新容器但共享内部对象的引用；深拷贝递归创建所有层级的独立副本。 2. 底层实现：copy.copy() 调用对象的 __copy__ 方法（列表用切片 [:]，字典用 dict.copy()）；copy.deepcopy() 递归遍历对象图，用 memo 字典处理循环引用。 3. 必须用深拷贝的场景：多维列表/嵌套字典的独立修改；对象树结构中子对象的隔离；并发环境下避免共享可变状态。 4. 常见坑点：深拷贝性能开销大（尤其是对象图复杂时）；某些不可拷贝对象（如文件句柄、锁）会导致异常；自定义类需要实现 __deepcopy__ 才能正确深拷贝。 5. 生产优化：优先用不可变数据结构避免拷贝；需要频繁拷贝时考虑 copy 模块的 __copy__ 自定义实现；大数据结构用 pickle 序列化/反序列化实现深拷贝。
