    Eval、Trace 与 Safety 真题与工程追问¶

    一、示范真题¶

    题 1：你改了 RAG Prompt，怎么证明效果真的提升¶

    回答骨架：

    1. 固定评测集和版本，区分有证据、无证据、边界问题。
    2. 保持检索策略不变，单独比较 Prompt 改动前后的答案指标。
    3. 同时看引用、faithfulness、拒答质量和延迟。
    4. 保存 Trace 与失败样本，避免只展示成功 case。

    题 2：一个工具型 Agent 执行成功但业务结果错了，怎么排查¶

    回答骨架：

    1. 工具成功不等于任务正确。
    2. 查模型是否选错工具、参数是否表达错意图、运行时是否放过越界参数。
    3. 查工具结果是否被模型误读。
    4. 把失败归入 taxonomy，补参数正确率和最终任务完成率 case。

    题 3：网页内容里藏了诱导指令，Agent 又能调高风险工具，怎么防¶

    回答骨架：

    1. 把外部内容视为不可信输入，不让它升级系统权限。
    2. 高风险工具只暴露最小能力，并在运行时做权限和参数策略。
    3. 重要动作先 preview，再人工确认。
    4. 记录内容来源、工具调用和策略决策，便于审计。

    二、工程追问清单¶

    追问 回答提示
    没有标注集怎么开始 Eval 先建高价值小集，持续收失败样本
    多轮 Agent 怎么算 Pass@1 先定义任务成功判定与允许步骤
    Trace 太大怎么办 记录关键 span、引用 ID、摘要和错误
    Safety 评测怎么做 加越权、注入、拒答和确认 case
    HITL 会拖慢体验怎么办 只放在副作用大或低置信动作

    三、复习路径¶

    1. 回 学习页 校准指标与边界。
    2. 对照 Harness 工程 看长任务执行环境。
    3. 用 高频八股 压短答案。
